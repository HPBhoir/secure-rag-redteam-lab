LLM Red-Teaming Lab - Useful Commands

========================================
1. ENVIRONMENT SETUP
========================================

Create environment:
python -m venv rag-env

Activate environment (macOS/Linux):
source rag-env/bin/activate

Install dependencies:
pip install -r requirements.txt

Optional manual install:
pip install streamlit langchain langchain-community langchain-chroma chromadb sentence-transformers ollama

========================================
2. PROJECT NAVIGATION
========================================

Go to project root:
cd /Users/harshalbhoir/Downloads/llm-redteam-lab

Check current branch:
git branch

See current status:
git status

========================================
3. RUN THE INGESTION PIPELINE
========================================

Run ingestion script:
python ingest.py

This loads documents from:
data/clean/
data/poisoned/

and stores embeddings in:
vectordb/

========================================
4. RUN THE STREAMLIT UI
========================================

From project root:
streamlit run app/ui.py

If already inside app/ folder:
streamlit run ui.py

========================================
5. OLLAMA COMMANDS
========================================

Check available models:
ollama list

Run model manually:
ollama run llama3

Make sure Ollama is running before using the chatbot.

========================================
6. GIT WORKFLOW
========================================

Initialize repo:
git init

Add all files:
git add .

Create initial vulnerable commit:
git commit -m "vulnerable baseline"

Rename branch to main (optional):
git branch -m main

Create hardening branch:
git checkout -b hardening

Switch back to vulnerable branch:
git checkout main

Switch back to hardened branch:
git checkout hardening

Commit hardened version:
git add .
git commit -m "add full RAG hardening: prompt, retrieval, tool, and output security"

Commit attack-documentation updates:
git add .
git commit -m "add data exfiltration attack results (before hardening)"

See commit history:
git log --oneline --graph --decorate --all

========================================
8. LOGS AND OUTPUTS
========================================

Logs are written to:
logs/

Each run creates a JSON log with:
- question
- retrieved chunks
- safe chunks
- flagged chunks
- tool result
- final prompt
- answer

Open latest logs:
ls -lt logs

Open specific log:
cat logs/<filename>.json

========================================
11. TROUBLESHOOTING
========================================

If 'No module named app':
- make sure app/__init__.py exists
- run from project root:
  streamlit run app/ui.py

If streamlit says file does not exist:
- from project root: streamlit run app/ui.py
- from app folder: streamlit run ui.py

If retrieval seems empty:
- run: python ingest.py
- confirm vectordb/ exists

If model does not answer:
- run: ollama list
- ensure llama3 is installed and available
